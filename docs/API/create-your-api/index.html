Напишем свой API!

Создание собственного API — это не только увлекательный, но и крайне полезный процесс, который позволяет организовать взаимодействие между различными приложениями и системами. В этом руководстве мы поэтапно разберем, как спроектировать и реализовать API, начиная с базовых принципов и заканчивая работающим приложением. Мы пройдем все ключевые шаги, чтобы создать API, который будет не только функциональным, но и легко поддерживаемым и расширяемым в будущем.

В рамках нашего проекта мы разработаем сервер для собачьего приюта, который будет обрабатывать запросы от клиента и предоставлять информацию о собаках, хранящихся в базе данных. Сервер будет поддерживать различные типы запросов, включая получение данных по идентификатору, породе, возрасту и имени собаки. Клиентская часть программы позволит пользователю делать запросы к серверу и получать нужную информацию в удобном формате.

Данная разработка будет имитировать передачу данных по протоколу JSON-RPC.

Шаг 1: Определение требований​

Прежде чем приступить к разработке, крайне важно четко сформулировать задачи, которые будет решать ваш API. Это поможет создать структуру, которая будет соответствовать потребностям пользователей и обеспечит легкость в дальнейшей поддержке и расширении. Рассмотрим ключевые аспекты, которые необходимо определить на этом этапе:

  • Цель API: Определите основную задачу, которую должен выполнять ваш API. Какую проблему он решает? Какие конкретные функции он должен предоставлять пользователям?
  • Целевая аудитория: Определите, кто будет использовать ваш API. Это могут быть разработчики, сотрудники приютов, сторонние системы или приложения. Понимание аудитории позволит лучше адаптировать функционал и интерфейс API под их потребности.
  • Функциональные требования: Опишите конкретные действия, которые должен выполнять API. Какие операции должны быть доступны пользователям? Какие данные будут обрабатываться?

Наш пример:

  • Цель: Разработка API, который будет обрабатывать клиентские запросы и предоставлять информацию с сервера о собаках, находящихся в приюте.
  • Целевая аудитория: Собачьи приюты и их сотрудники, желающие автоматизировать процесс предоставления информации и уменьшить нагрузку на операторов колл-центров.
  • Функциональные требования: API должен предоставлять возможность получения информации о собаках по ID, породе, возрасту и имени, без возможности редактирования данных в базе.

Шаг 2: Проектирование API​

На этапе проектирования API важно разработать его структуру и определить ключевые компоненты, которые будут взаимодействовать с пользователями. Это включает:

  • Определение конечных точек (endpoints): Решите, какие URL-адреса будут доступны для взаимодействия с API и какие действия будут выполняться при обращении к ним.
  • Методы HTTP: Определите, какие методы HTTP (например, GET, POST) будут использоваться для каждой конечной точки. Это позволит задать контекст для запросов: получение данных, создание новых записей или выполнение других операций.
  • Формат данных: Определите формат данных, в котором будут передаваться запросы и ответы. В нашем проекте используется JSON как стандартный формат для обмена данными между клиентом и сервером.
  • Структура ответа и запроса: Определите, какие поля будут присутствовать в запросах и ответах API. Убедитесь, что структура данных понятна и согласована с функциональными требованиями.

Пример проектирования для нашего API:

  1. Конечные точки:
  • /jsonrpc (POST) - единственная конечная точка, которая обрабатывает различные типы запросов с использованием JSON-RPC. Конкретное действие определяется параметром method в теле запроса.
  1. Методы HTTP:
  • POST (POST) - используется для отправки запросов на сервер с различными методами, такими как получение информации о собаках по ID, породе, возрасту или имени.
  1. Формат данных:
  • JSON - как для запросов, так и для ответов. JSON прост в использовании и широко поддерживается, что делает его идеальным выбором для API.

Шаг 3: Реализация API​

Теперь, когда вы спроектировали API, можно перейти к его реализации. В этом разделе мы покажем, как создать простой API на C++ с использованием JSON-RPC.

Наша программа будет имитировать работу сервера приюта для собак. Данные о собаках будут храниться на сервере в виде JSON-массива. Клиентское приложение сможет отправлять запросы на сервер и получать информацию о собаках в соответствии с запросами. Мы реализуем два ключевых компонента:

  1. Сервер: Будет реализован на C++ с использованием библиотеки httplib. Сервер будет хранить информацию о собаках в формате JSON и обрабатывать входящие запросы JSON-RPC. Он будет поддерживать несколько методов запроса, таких как получение информации о собаке по ID, породе, возрасту или имени.
  2. Клиент: Также реализованный на C++ с использованием библиотеки cpr. Клиент будет отправлять запросы на сервер, получать ответы и выводить информацию о собаках в зависимости от типа запроса. Клиент сможет делать запросы по ID, породе, возрасту или имени, а также иметь возможность выйти из программы по запросу пользователя.

Пример на C++ с JSON-RPC​

Для создания API на C++ нам понадобятся следующие библиотеки:

  • nlohmann/json - для работы с JSON.
  • cpp-httplib - для создания простого HTTP сервера.
  • cpr - для выполнения HTTP запросов (в клиентской части).

Серверная часть на C++
#include <iostream>
#include "httplib.h"
#include "json.hpp"

using json = nlohmann::json;
using namespace httplib;

// Функция для получения данных о собаках, которые хранятся непосредственно в коде.
// Возвращает JSON-массив с информацией о собаках.
// Это должно быть в отдельном файле, но у меня возникли какие-то проблемы с потоками, поэтому все здесь
json get_dogs_data() {
    return json::array({
                               {{"id", 1}, {"name", "Rex"}, {"breed", "German Shepherd"}, {"age", 3}},
                               {{"id", 2}, {"name", "Bella"}, {"breed", "Labrador"}, {"age", 5}},
                               {{"id", 3}, {"name", "Charlie"}, {"breed", "Beagle"}, {"age", 2}},
                               {{"id", 4}, {"name", "Luna"}, {"breed", "Poodle"}, {"age", 4}},
                               {{"id", 5}, {"name", "Max"}, {"breed", "Bulldog"}, {"age", 6}},
                               {{"id", 6}, {"name", "Lucy"}, {"breed", "Boxer"}, {"age", 1}},
                               {{"id", 7}, {"name", "Bailey"}, {"breed", "Dalmatian"}, {"age", 3}},
                               {{"id", 8}, {"name", "Daisy"}, {"breed", "Golden Retriever"}, {"age", 2}},
                               {{"id", 9}, {"name", "Buddy"}, {"breed", "Husky"}, {"age", 4}},
                               {{"id", 10}, {"name", "Molly"}, {"breed", "Shih Tzu"}, {"age", 7}}
                       });
}

// Функция для обработки входящих JSON-RPC запросов.
// Принимает запрос в формате JSON и данные о собаках, возвращает JSON-ответ.
json handle_request(const json& request, const json& dogs_data) {
    // Извлекаем метод из запроса.
    std::string method = request.value("method", "");
    json response;

    // Если метод запроса - получение собаки по ID
    if (method == "get_dog_by_id") {
        // Получаем ID из параметров запроса
        int id = request["params"].value("id", 0);

        // Ищем собаку с данным ID
        for (const auto& dog : dogs_data) {
            if (dog["id"] == id) {
                // Если собака найдена, формируем ответ с результатом
                response = {
                        {"jsonrpc", "2.0"},
                        {"result", dog},
                        {"id", request["id"]}
                };
                return response;
            }
        }

        // Если собака не найдена, возвращаем ошибку
        response = {
                {"jsonrpc", "2.0"},
                {"error", {
                                    {"code", -32602},
                                    {"message", "Dog not found"}
                            }},
                {"id", request["id"]}
        };

        // Если метод запроса - получение собак по породе
    } else if (method == "get_dogs_by_breed") {
        // Получаем породу из параметров запроса
        std::string breed = request["params"].value("breed", "");
        json result = json::array();

        // Ищем всех собак с данной породой
        for (const auto& dog : dogs_data) {
            if (dog["breed"] == breed) {
                result.push_back(dog);
            }
        }

        // Формируем ответ с результатом
        response = {
                {"jsonrpc", "2.0"},
                {"result", result},
                {"id", request["id"]}
        };

        // Если метод запроса - получение собак по возрасту
    } else if (method == "get_dogs_by_age") {
        // Получаем возраст из параметров запроса
        int age = request["params"].value("age", 0);
        json result = json::array();

        // Ищем всех собак с данным возрастом
        for (const auto& dog : dogs_data) {
            if (dog["age"] == age) {
                result.push_back(dog);
            }
        }

        // Формируем ответ с результатом
        response = {
                {"jsonrpc", "2.0"},
                {"result", result},
                {"id", request["id"]}
        };

        // Если метод запроса - получение собаки по имени (новый метод)
    } else if (method == "get_dog_by_name") {
        // Получаем имя из параметров запроса
        std::string name = request["params"].value("name", "");

        // Ищем собаку с данным именем
        for (const auto& dog : dogs_data) {
            if (dog["name"] == name) {
                // Если собака найдена, формируем ответ с результатом
                response = {
                        {"jsonrpc", "2.0"},
                        {"result", dog},
                        {"id", request["id"]}
                };
                return response;
            }
        }

        // Если собака не найдена, возвращаем ошибку
        response = {
                {"jsonrpc", "2.0"},
                {"error", {
                                    {"code", -32602},
                                    {"message", "Dog not found"}
                            }},
                {"id", request["id"]}
        };

        // Если запрашивается неизвестный метод
    } else {
        response = {
                {"jsonrpc", "2.0"},
                {"error", {
                                    {"code", -32601},
                                    {"message", "Method not found"}
                            }},
                {"id", request["id"]}
        };
    }

    // Возвращаем ответ на запрос
    return response;
}

int main() {
    // Создаем объект сервера
    Server svr;

    // Загружаем данные о собаках в память
    json dogs_data = get_dogs_data();

    // Устанавливаем обработчик для POST-запросов на /jsonrpc
    svr.Post("/jsonrpc", [&dogs_data](const Request& req, Response& res) {
        std::cout << "Received request: " << req.body << std::endl;

        json request;

        try {
            // Парсим тело запроса как JSON
            request = json::parse(req.body);
        } catch (const json::parse_error& e) {
            // Обрабатываем ошибку парсинга JSON
            std::cerr << "Parse error: " << e.what() << std::endl;
            res.status = 400;
            res.set_content("{\"jsonrpc\": \"2.0\", \"error\": {\"code\": -32700, \"message\": \"Parse error\"}, \"id\": null}", "application/json");
            return;
        }

        // Обрабатываем запрос и получаем ответ
        json response = handle_request(request, dogs_data);

        // Выводим и отправляем ответ
        std::cout << "Sending response: " << response.dump() << std::endl;
        res.set_content(response.dump(), "application/json");
    });

    // Запускаем сервер на localhost:8080
    std::cout << "Server started at http://localhost:8080\n";
    svr.listen("localhost", 8080);

    return 0;
}


Клиентская часть на C++
#include <iostream>
#include <cpr/cpr.h>
#include "json.hpp"

using json = nlohmann::json;

// Перечисление для типа запроса
// Используется для повышения читаемости и поддерживаемости кода
enum class RequestType {
    GetDogById = 1,    // Получение собаки по ID
    GetDogsByBreed,    // Получение собак по породе
    GetDogsByAge,      // Получение собак по возрасту
    GetDogByName,      // Получение собаки по имени
    Exit = 0           // Выход из программы
};

int main() {
    int option;

    // Бесконечный цикл, продолжающий работу программы, пока не будет выбран выход
    while (true) {
        // Предлагаем пользователю выбрать тип запроса
        std::cout << "Choose request type:\n";
        std::cout << "1. Get dog by ID\n";
        std::cout << "2. Get dogs by breed\n";
        std::cout << "3. Get dogs by age\n";
        std::cout << "4. Get dog by name\n"; // Добавлен новый вариант запроса по имени
        std::cout << "0. Exit\n";            // Опция для выхода
        std::cout << "Enter option: ";
        std::cin >> option;

        // Преобразуем введенный пользователем вариант в тип RequestType
        RequestType request_type = static_cast<RequestType>(option);

        // Если пользователь выбрал выход, завершаем программу
        if (request_type == RequestType::Exit) {
            std::cout << "Exiting the program.\n";
            break;
        }

        json request; // Создаем JSON-объект для хранения данных запроса

        // В зависимости от выбранного типа запроса, формируем соответствующий JSON-RPC запрос
        switch (request_type) {
            case RequestType::GetDogById: {
                int id;
                std::cout << "Enter dog ID: ";
                std::cin >> id;

                // Формируем запрос для получения собаки по ID
                request = {
                        {"jsonrpc", "2.0"},
                        {"method", "get_dog_by_id"},
                        {"params", {{"id", id}}},
                        {"id", 1}
                };
                break;
            }
            case RequestType::GetDogsByBreed: {
                std::string breed;
                std::cout << "Enter dog breed: ";
                std::cin >> breed;

                // Формируем запрос для получения собак по породе
                request = {
                        {"jsonrpc", "2.0"},
                        {"method", "get_dogs_by_breed"},
                        {"params", {{"breed", breed}}},
                        {"id", 2}
                };
                break;
            }
            case RequestType::GetDogsByAge: {
                int age;
                std::cout << "Enter dog age: ";
                std::cin >> age;

                // Формируем запрос для получения собак по возрасту
                request = {
                        {"jsonrpc", "2.0"},
                        {"method", "get_dogs_by_age"},
                        {"params", {{"age", age}}},
                        {"id", 3}
                };
                break;
            }
            case RequestType::GetDogByName: { // Новый случай для запроса по имени
                std::string name;
                std::cout << "Enter dog name: ";
                std::cin >> name;

                // Формируем запрос для получения собаки по имени
                request = {
                        {"jsonrpc", "2.0"},
                        {"method", "get_dog_by_name"},
                        {"params", {{"name", name}}},
                        {"id", 4}
                };
                break;
            }
            default: {
                std::cout << "Invalid option selected.\n";
                continue; // Возвращаемся к началу цикла, если выбран неверный вариант
            }
        }

        // Сериализация JSON-объекта в строку для отправки запроса на сервер
        std::string request_body = request.dump();
        std::cout << "Request body: " << request_body << '\n';

        // Отправляем POST-запрос на локальный сервер
        cpr::Response r = cpr::Post(cpr::Url{"http://localhost:8080/jsonrpc"},
                                    cpr::Body{request_body},
                                    cpr::Header{{"Content-Type", "application/json"}});

        // Проверка на ошибки HTTP-запроса
        if (r.error) {
            std::cerr << "Error during request: " << r.error.message << '\n';
            continue; // Возвращаемся к началу цикла в случае ошибки
        }

        // Выводим статус-код и тело ответа
        std::cout << "Status code: " << r.status_code << '\n';
        std::cout << "Response body: " << r.text << '\n';

        // Обработка ответа сервера
        try {
            if (!r.text.empty()) {
                json response = json::parse(r.text); // Парсим ответ сервера в JSON

                // Проверяем, содержит ли ответ ошибку
                if (response.contains("error")) {
                    std::cout << "Error: " << response["error"]["message"] << '\n';
                } else {
                    std::cout << "Result: " << response["result"] << '\n'; // Выводим результат
                }
            } else {
                std::cerr << "Empty response body received from server.\n";
            }
        } catch (const json::parse_error& e) {
            std::cerr << "Parse error: " << e.what() << '\n'; // Обработка ошибок парсинга JSON
        }
    }

    return 0; // Завершение программы
}

Компиляция и запуск​

  1. Компиляция сервера и клиента:

    Используйте сборщик проектов или интегрированную среду разработки (IDE) по вашему выбору для компиляции исходных файлов сервера и клиента. Убедитесь, что все необходимые зависимости и библиотеки подключены.

  2. Запуск сервера:

    Запустите скомпилированный серверный исполняемый файл. Сервер начнет прослушивать входящие запросы и будет готов обрабатывать их

  3. Запуск клиента:

    Запустите скомпилированный клиентский исполняемый файл. Клиент отправит запрос на сервер в соответствии с выбранным пунктом меню и получит ответ с информацией о запрашиваемой собаке.

Клиент взаимодействует с сервером через JSON-RPC запросы и получает данные о собаках, которые были заранее определены на сервере. Убедитесь, что сервер запущен до запуска клиента, чтобы обеспечить успешное взаимодействие.
